# TwiCC Activity Changelogs

Generate Markdown changelogs for Git repositories that had recent TwiCC session
activity.

The script asks TwiCC for recent sessions, maps those sessions back to local Git
repositories, collects recent Git context for each repository, then asks the
selected generator, Codex or Claude, to write one concise changelog per project.

## Requirements

- `twicc` available on `PATH`, or passed with `--twicc-bin`.
- TwiCC with provider passthroughs available. By default the script invokes
  `twicc codex` for Codex and `twicc claude` for Claude, so standalone provider
  CLIs do not need to be installed separately.
- `git` available on `PATH`.
- Local repositories referenced by TwiCC sessions must still exist on disk.

## Usage

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs --clean
```

This generates:

- one Markdown changelog per active repository
- an `index.md` linking to all generated changelogs
- an `_inputs/` directory containing the prompts sent to the selected generator

The first argument is the activity window. Supported formats are:

- `24h`
- `7d`

The second argument is the output directory.

## Common Commands

Generate changelogs for the last 7 days:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs --clean
```

Preview what would be generated without calling the LLM generator:

```bash
./twicc-activity-changelogs.py 24h /tmp/my-changelogs --dry-run --clean
```

Use Claude instead of Codex:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs \
  --generator claude \
  --clean
```

Use a specific Claude model:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs \
  --generator claude \
  --claude-model sonnet \
  --clean
```

Limit activity discovery to one TwiCC project:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs --project /path/to/project --clean
```

Limit activity discovery to one TwiCC workspace:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs --workspace <workspace-id> --clean
```

Keep a copy of the discovered active repository list:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs --repos-json /tmp/my-changelogs/repos.json --clean
```

Only include repositories matching one or more regular expressions:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs \
  --whitelist 'github.com/my-org/' \
  --whitelist 'important-project' \
  --clean
```

Exclude repositories matching one or more regular expressions:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs \
  --blacklist 'archive|experiment' \
  --clean
```

Only include repositories whose remote is publicly readable without credentials:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs --public --clean
```

## Output Cleanup

By default, the output directory is reused as-is. Existing files can remain from
previous runs if a repository is no longer active or if fewer changelogs are
generated.

Pass `--clean` to remove files generated by previous runs before generating new
ones. It removes:

- `*.md` files at the root of the output directory
- the `_inputs/` directory

Other files in the output directory are left untouched.

## Repository Selection

A repository is considered active when a recent TwiCC session points to a local
path inside that Git repository. The script checks these session fields:

- `git_directory`
- `cwd`

The session activity timestamp is taken from the first available field among:

- `last_new_content_at`
- `last_updated_at`
- `last_started_at`
- `created_at`
- `mtime`

Hidden and archived sessions are excluded by default. Use `--include-hidden` and
`--include-archived` to include them.

After active repositories are discovered, additional repository filters can be
applied:

- `--whitelist REGEX` only keeps matching repositories.
- `--blacklist REGEX` removes matching repositories.
- `--public` only keeps repositories whose remote can be read without
  credentials.

`--whitelist` and `--blacklist` are repeatable. They match against the repository
name, local path, remote URL, TwiCC project IDs, and branches seen in TwiCC
sessions. `--allowlist` and `--denylist` are accepted as aliases.

`--public` checks each remote with `git ls-remote` while disabling interactive
credential prompts. SSH-style remotes such as `git@github.com:org/repo.git` are
probed through their HTTPS equivalent when possible. Repositories with no remote,
private remotes, unreachable remotes, or remotes that time out are skipped.
Adjust the per-repository timeout with `--public-timeout`.

## Git Context

For each active repository, the script collects:

- current branch and HEAD
- TwiCC branches seen in recent sessions
- matching commits in the requested window
- committed change summaries
- changed files
- working tree status
- unstaged and staged diff stats

By default, commits are filtered to the repository's configured Git author:

- `git config user.email`
- `git config user.name`

Use `--author` to override this. It accepts a regular expression and can be
provided multiple times:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs \
  --author 'alice@example.com' \
  --author 'Alice Doe' \
  --clean
```

Uncommitted local changes, including untracked files visible in `git status`, are
enough for a repository to get a changelog. Repositories with no matching commits
or local changes are skipped unless `--include-empty` is passed.

## Generator Prompt Inputs

The generated `_inputs/*.txt` files contain the exact context and instructions
sent to the selected generator. They are useful for debugging, reviewing, or
rerunning a single generation manually.

These files can contain local paths, repository names, branch names, Git author
information, commit subjects, changed file names, and working tree status. Review
them before sharing generated output directories publicly.

By default, committed changes are summarized with metadata, changed file names,
and diff stats, but the full patch text is not sent to the generator. This keeps
prompts smaller and avoids including detailed code diffs unless explicitly
requested.

Use `--include-patch` to include committed patch text in the prompt inputs:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs --include-patch --clean
```

This can improve changelog quality when commit subjects and stats are not
descriptive enough, but it may expose more source code in `_inputs/*.txt` and to
the selected generator. Patch text is truncated per repository. Adjust the limit
with `--max-patch-chars`.

## Environment Variables

The script can also read command overrides from environment variables:

- `TWICC_BIN`
- `CODEX_BIN`
- `CLAUDE_BIN`

By default, Codex generation uses `twicc codex` and Claude generation uses
`twicc claude`. Command-line options take precedence when you want to use a
standalone CLI or a custom wrapper:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs \
  --twicc-bin /path/to/twicc \
  --codex-bin /path/to/codex \
  --clean
```

With Claude:

```bash
./twicc-activity-changelogs.py 7d /tmp/my-changelogs \
  --generator claude \
  --claude-bin /path/to/claude \
  --clean
```

Claude is invoked in non-interactive print mode with tools disabled. The full
prompt is passed through stdin, and stdout is written to the changelog file.
